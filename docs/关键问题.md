解决：
接修改内置工具 builtin_tools.py：当用户给出无扩展名路径（如 README）且文件不存在时，自动在同目录按同名 stem 匹配（优先 .md），这样“README”和“README.md”都能成功。随后我会实际跑你给的命令验证。

"模型调用失败。请检查 OPENAI_BASE_URL 是否兼容 OpenAI Chat Completions 工具调用协议，原始错误: StructuredTool does not support sync invocation.",
我确认了根因：--enable-mcp 时加载到的是异步 MCP 工具，而 AgentService.ask 走同步 invoke，所以当模型真正调用这些工具就报错。现在我会把 service.py 改为异步 ainvoke，并同步调整 app.py 的调用。
MCP 工具很多是“仅异步”的 StructuredTool
如果调用链里有人走了同步入口（比如 invoke），LangChain 会尝试同步执行工具
仅异步工具没有同步实现，就会抛出：
StructuredTool does not support sync invocation
为什么“读取前10行”常常能过：

这个问题更容易命中你自己的内置读文件工具，而不是 MCP 工具
内置工具通常可同步执行，所以不会触发上述异常
“读取 README 并总结”更开放，模型更可能选 MCP 文件工具，于是暴露同步/异步不匹配
